Docker初识

发布于 2020-02-02 | 分类于 容器技术 | 9分钟 | 2253字数

Em...简单地聊下容器技术与Docker...

一、虚拟化技术与容器化技术

虚拟化

先来简单了解下虚拟化吧，虚拟机大家应该都不会陌生吧，以前用windows主机，想玩Linux的时候，装双系统又嫌麻烦，通常会选择装个VMware或者Virtualbox，然后在上面创建Linux虚拟机。虚拟机便是虚拟化技术的一种实现，来看下VMware官方对虚拟化的解释：

虚拟化是为一些组件（例如虚拟应用、服务器、存储和网络）创建基于软件的（或虚拟）表现形式的过程。

通俗的讲就是通过软件手段将一些硬件资源（如CPU资源、存储资源）“虚拟”出来。

那么，为什么会有虚拟化技术呢？或者它是解决什么问题的？

还记得前文提到的Linux虚拟机么，如果没有虚拟化技术，我们在windows主机上想要安装Linux系统基本只能通过装双系统的方式实现，但是装双系统有个缺陷就是同一时刻你只能使用其中一个系统，不能在两个系统间随意切换，而且还需要考虑硬件兼容性问题。如此，虚拟化技术的一个好处就体现出来了：实现程序跨平台运行。

除此之外，虚拟化还有一个优势在于资源管理。用过VMware的同学都知道我们可以通过VMware为我们创建的Linux虚拟机设置CPU核心数、内存大小等等，通过这些配置我们可以灵活的管控Linux虚拟机所需的资源。

虚拟化技术在实际场景中有很多种分类，包括平台虚拟化、应用程序虚拟化等等，这不是这片文章的重点，网上资料也很多，这里就不再逐一介绍了（其实是懒了！）。

容器化

容器化又是什么呢？和虚拟化有关系么？为什么会有容器化呢？

其实容器化是一种轻量级虚拟化技术，“轻”在哪里？前文提到的虚拟化技术-虚拟机是基于系统为中心实现的，而容器化则是以应用程序为中心的。容器这个词非常形象了，既然是“容器”，那它肯定能装东西喽，既然容器化是以应用程序为中心，那肯定是“装”应用程序了。

其实容器技术是将一个应用程序及其所需的运行环境都打包起来变成一个沙盒“容器”，并且不同“容器”间互不干扰，独立运行。

容器化技术不仅部署应用方便、隔离性好，而且在应用运行性能上要远超虚拟机等其他虚拟化实现。

关于更多容器化的知识可参考Docker what-container。

虚拟机? OR 容器?

在一篇博客（文末有链接）上看到一张图，可以很直观地看出二者的区别及优劣了：

二者最主要的区别其实就是虚拟机中的程序是运行在虚拟机操作系统中的，而容器中的程序则是完全运行在宿主机上的，就像你主机上目前运行的程序一样。

二、Docker

Docker这个词我第一次听到应该是大四实习的时候，那个时候给我印象有点深的是它的logo，很cute!! 之后工作中一直没有什么机会接触这块知识，正好最近有一次机会，一起来简单地学习下吧！咱么日常三问开头，What? Why? How?

What's Docker?

来看看Docker官方对Docker的解释：

Docker is an open platform for developing, shipping, and running applications. Docker enables you to separate your applications from your infrastructure so you can deliver software quickly.

Docker官方将Docker定义为一个开放平台，用于开发、交付以及运行应用程序。上文不是提到了容器化嘛？Docker一般不也被叫做Docker容器么，所以docker其实就是容器化技术的一种实现，我们完全可以通过Docker来部署执行我们的应用程序。

Why Docker?

我们来看下Docker有什么特性及优势：

  • 快速部署：通过Docker容器可快速部署运行你的应用。
  • 隔离性和安全性：使用Docker可以进行资源隔离，确保每个应用程序只使用分配给它的资源，并且多个Docker容器之间互不影响。
  • 持续集成和交付：定制应用镜像来实现集成、持续支付、部署。开发人员可以通过dockerfile来进行镜像构建，并结合持续集成系统进行集成测试，而运维人员则可以直接在生产环境中快速部署该镜像，甚至结合持续部署系统进行自动部署。
  • 多平台快速迁移：由于docker确保了执行环境的一致性，使得应用的迁移更加的容易。docker可以在很多平台上运行，无论是物理机、虚拟机、公有云、私有云、甚至是笔记本、其运行结果是一致的。因此用户可以很轻易的将在一个平台上运行的应用，迁移到另一个平台上，而不用担心运行环境的变化导致应用无法正常运行的情况。

单看这些优势是不是有点懵？我们后面会用Docker简单实操一遍，便于理解它的优势。

Docker架构

Docker采用了CS架构，包括如下几个部分：

1.Docker daemon

守护进程（dockered），负责处理Client端的请求并管理Docker对象，例如镜像、容器、网络等等。

2.Docker client

Docker客户端，是用户与Docker交互的桥梁，当用户执行docker run之类的命令时，客户端将命令发送至dockered，交由dockered执行。

3.Docker registry

Docker注册表，代表Docker镜像存储库，默认注册表是 Docker Hub，支持配置其他注册表。

4.Docker objects

  • images：镜像，一个只读的文件系统，包含容器运行时所需要的文件和配置参数。通过镜像来创建container，并且一个镜像可以运行多个container。镜像可以从仓库中下载，也可以用Dockerfile自行创建。
  • containers：容器，是images的一个可运行实例，可以使用Docker客户端创建，启动，停止，移动或删除容器。
  • repository：仓库，有点类似于上文的registry，就是Docker存放镜像文件的地方，可类比成Maven的仓库。当然Docker也有公有仓库和私有仓库。

Docker的基本使用

下文以在Docker中安装运行Tomcat为例介绍下Docker的基本使用方式：

1.Docker安装

注意Docker安装的前提需要操作系统支持虚拟化！

windows：win7和win8需通过docker toolbox安装，win10系统需开启虚拟化（Hyper-V）后通过Docker Desktop for Windows安装

macos：通过Docker Desktop for Mac安装

linux：通过Docker for Linux安装

2.配置镜像加速

4.运行容器

有了镜像之后，我们可以直接通过如下命令创建容器并运行：

docker run --name tomcat -p 8080:8080 -v /$PWD/test:/usr/local/tomcat/webapps/test -d tomcat

  • --name：指定容器name
  • -p：Docker容器端口和宿主机端口映射
  • -v：将主机中当前目录下的 test 挂载到容器的 /test
  • -d：指定镜像

执行成功后，会返回一个容器ID，我们可以通过容器ID或指定的name查看启动日志：

docker logs da3888776b2cd5dc6ed26fb64c9f5c8a9a8002b5225f840a5e7beef424b75c04
